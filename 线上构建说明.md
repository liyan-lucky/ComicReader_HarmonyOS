# 线上构建说明

本文说明漫画浏览器 App 的 GitHub Actions 在线构建流程。

## 构建入口

进入 App 仓库：

```text
Actions → 漫画浏览器 Linux 在线构建 → Run workflow
```

这个流程用于测试项目结构、SDK 配置、hvigor 构建链路，并尝试生成 HAP 包。

## 你需要添加的两个对象

进入仓库：

```text
Settings → Secrets and variables → Actions
```

可以添加到 `Variables`，也可以添加到 `Secrets`。如果链接不想公开，建议放到 `Secrets`。

### 1. 完整 SDK 包链接

对象名：

```text
HARMONYOS_SDK_URL
```

用途：

```text
完整 HarmonyOS SDK 压缩包下载链接。
```

压缩包解开后，流程会自动寻找这些内容：

```text
openharmony
hms
sdk-pkg.json
command-line-tools
```

只要这些目录不在根目录，流程也会尝试在子目录里自动寻找。

### 2. SDK 以外补充文件链接

对象名：

```text
HARMONYOS_FULL_URL
```

用途：

```text
SDK 以外的完整命令行工具、hvigor、ohpm 或补充文件压缩包下载链接。
```

这个对象是为了解决 GitHub Linux 环境里命令行工具不完整的问题。它会被合并到：

```text
/home/runner/harmonyos-sdk/command-line-tools
```

## 流程做了什么

```text
1. 拉取源码
2. 准备 Java 17
3. 准备 Node 20
4. 安装 unzip、zip、curl、jq、python3、rsync 等工具
5. 检查 HARMONYOS_SDK_URL 和 HARMONYOS_FULL_URL 是否已填写
6. 下载并解压完整 SDK
7. 下载并合并 SDK 以外补充文件
8. 检查 App 项目结构
9. 安装 ohpm 依赖
10. 临时改成未签名构建
11. 执行 HAP 构建
12. 上传构建产物
```

## 为什么先做未签名构建

当前流程主要用于验证结构和构建链路，不先卡签名。它会临时把：

```text
signingConfig
```

调整为空，先尝试生成未签名 HAP。后续如果需要正式安装包，再追加签名材料流程。

## 成功后在哪里看包

运行成功后，在本次工作流页面底部查看：

```text
Artifacts
```

产物名称：

```text
漫画浏览器-HAP-构建产物
```

## 如果失败怎么看原因

优先看这些步骤：

```text
检查必须填写的两个链接
下载并安装完整 SDK
下载并合并 SDK 以外补充文件
检查项目结构
执行 HAP 构建
```

常见问题：

```text
没有填写 HARMONYOS_SDK_URL：说明第一个对象没填。
没有填写 HARMONYOS_FULL_URL：说明第二个对象没填。
没有找到 SDK 根目录：说明 SDK 压缩包层级或内容不对。
没有找到 ohpm：说明补充工具包里缺少 ohpm，或路径不对。
HAP 构建命令没有成功：说明 hvigor 工具不完整，或项目配置仍有结构问题。
没有找到 HAP 包：说明构建返回成功但没有实际出包，需要看 hvigor 输出。
```
