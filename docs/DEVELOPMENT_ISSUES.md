# 开发问题记录

本文档记录开发过程中遇到的问题、原因、处理方式和当前状态。后续遇到同类问题时先查本文档。

## 当前原则

- 当前长期分支以 `main` 为准。
- 风险较大的改动应从 `main` 新建 `feature/*` 或 `fix/*` 分支。
- 不再使用自动 UI 注入脚本、自动 patch 脚本或 UI 自动应用 workflow。
- ArkTS 源码修改优先直接替换完整文件，避免片段注入导致状态不可控。
- 重要节点使用 `backup/*` 分支或备份 workflow 保存回滚点。
- 根目录只保留项目入口、许可证、构建系统必需文件和工具必须识别的配置；长期文档统一归入 `docs/`。

## 问题记录

### 1. 搜索页混入设置入口

- 现象：搜索页曾出现语言、主题、搜索模式、去设置引擎等设置类入口。
- 原因：早期把配置入口直接放在搜索主流程中。
- 处理：搜索页只保留搜索主流程；设置类功能迁移到设置页。
- 当前状态：已调整，后续继续保持搜索页简洁。

### 2. 设置页交互结构不清晰

- 现象：设置曾使用单独二级页面，用户希望不要跳转二级页。
- 正确需求：基础设置和 APP 设置直接展开在设置页；搜索菜单、源开关、显示与阅读、搜索引擎、规则设置、高级规则使用弹出对话框。
- 关于信息：直接展开在设置页，只保留版本号和构建时间。
- 当前状态：已按最新需求调整。

### 3. 底部 Tab 样式被误改

- 现象：一次替换 `Index.ets` 时把底部 Tab 改成了文字符号样式，偏离原风格。
- 正确需求：Tab 样式不应大改；保留图标 + 文字，选中绿色，未选灰色。
- 当前状态：已恢复为图标式 Tab。

### 4. 搜索按钮加载效果理解偏差

- 现象：曾在搜索按钮内部加入动态图标。
- 正确需求：搜索中时按钮外边框显示可转动的加载效果，不是在按钮文字内部加动态图标。
- 当前状态：已改为外层 LoadingProgress + 按钮显示“搜索中”。

### 5. 搜索“斗罗 / 斗罗大陆”混入抖音等无关结果

- 现象：搜索“斗罗”或“斗罗大陆”时出现抖音、短视频、百科、社区等非漫画结果。
- 原因：搜索引擎结果过滤过宽，只要标题或 URL 含关键词就可能进入结果。
- 处理：`SearchEngines.ets` 加入负面域名和负面词过滤，并对“斗罗”加入 `斗罗大陆 / Soul Land / Douluo Dalu` 别名扩展。
- 当前状态：已修复，仍需继续收集漏网域名。

### 6. 搜索状态被旧阅读状态污染

- 现象：搜索结果页曾显示旧阅读状态，例如“卷轴生成完成：120 张图片”。
- 原因：新搜索没有完全清空阅读上下文状态。
- 处理：新搜索开始时清空 `currentTitle`、`currentUrl`、`chapters`、`images` 等状态，并使用 `searchTitle` 固定显示当前搜索词。
- 当前状态：已修复。

### 7. APP 运行 2–3 分钟后崩溃

- 现象：单 APP 运行一段时间后自动崩溃。
- 高风险原因：自动补封面、阅读图片加载、分页追踪、图片状态重复刷新造成内存和状态刷新压力。
- 处理：默认关闭自动补封面；降低搜索结果和封面补全上限；降低分页上限；图片合并加软上限；`markImage()` 状态不变时不再重建数组。
- 当前状态：已做稳定性修复，用户测试暂时无问题。

### 8. 自动注入脚本导致本地编译问题

- 现象：通过脚本注入/局部 patch 修改 ArkTS 文件后，本地编译问题较多，代码状态不可控。
- 原因：自动脚本依赖文本锚点，容易残留旧 Builder、重复 helper、状态不一致。
- 处理：删除自动 UI 注入脚本、旧 patch 脚本和相关 workflow；后续源码修改直接替换完整文件。
- 当前状态：已清理，禁止重新引入。

### 9. 仓库文件散落缺少约束

- 现象：文档、脚本、workflow、临时说明容易散落在根目录或多个目录。
- 处理：更新 `REPOSITORY_STANDARDS.md`，新增 `FILE_PLACEMENT_RULES.md`，明确文件放置位置、禁止新增临时脚本和自动 UI 注入流程。
- 当前状态：已建立规范，后续新增文件必须按规范归类。

### 10. 根目录外侧文档未整理

- 现象：根目录仍保留 `BUILDING.md`、`COMPLIANCE.md`、`PRIVACY.md`、`SECURITY.md`、`RELEASE_CHECKLIST.md`、`MAINTAINERS.md` 等长期文档。
- 原因：前一轮只建立规范，没有实际迁移根目录文档。
- 处理：把构建、发布、维护、合规、隐私、安全、第三方来源、版权和贡献说明迁入 `docs/` 对应分类，并更新 README 和 docs 索引；根目录只保留入口和构建系统必需文件。
- 当前状态：已完成。

### 11. `Index.ets` 单文件过大

- 现象：页面、状态、搜索、阅读、设置全部集中在一个文件，在线修改和本地排错困难。
- 原因：快速迭代阶段未拆分页面和组件。
- 建议：后续按搜索、设置、阅读、书架/历史逐步拆分，但每次拆分范围要小，并保证构建通过。
- 当前状态：待重构。

### 12. 规则仓库和 App 主仓库职责需要保持清晰

- App 主仓库：放内置兜底规则、远程规则加载逻辑、搜索结果过滤逻辑和模型。
- 规则仓库：放大量站点规则、生成后的 `generated/index.json`、规则测试和规则文档。
- 当前状态：已记录到文件放置规范。

### 13. 备份分支使用需要规范

- 现象：主分支多次因为测试、回退、合并需要备份点。
- 处理：使用 `backup/*` 保存关键节点；新增手动 workflow，将当前 `main` 强制覆盖到 `backup` 分支作为快照。
- 当前状态：已完成。
