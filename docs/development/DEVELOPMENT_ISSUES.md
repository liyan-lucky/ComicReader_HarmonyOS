# 开发问题记录

本文档记录开发过程中遇到的问题、原因、处理方式和状态。后续遇到同类问题时先查本文档。

## 问题记录

### 1. 搜索页混入设置入口

- 现象：搜索页出现语言、主题、搜索模式、去设置引擎等设置类入口。
- 原因：早期把配置入口直接放在搜索主流程中。
- 处理：搜索页只保留搜索；设置类功能迁移到设置页。
- 状态：已规划并部分修复，继续完善。

### 2. 用户需求被误解为删除设置

- 现象：曾误以为“主页面不要有设置选项”是删除设置 Tab。
- 正确需求：搜索页不出现设置入口；设置 Tab 保留。
- 处理：保留设置功能和设置 Tab，仅移除搜索页设置类入口。
- 状态：已修正。

### 3. 阅读页悬浮工具挤在屏幕中间

- 现象：阅读页顶部信息条和底部按钮条叠在中间。
- 原因：`Stack()` 中悬浮控件未设置 `Alignment.Top` / `Alignment.Bottom`。
- 处理：顶部信息固定顶部，返回/书架/刷新固定底部。
- 状态：已修复并合并。

### 4. 搜索无结果

- 现象：安装后搜索没有结果。
- 可能原因：默认引擎依赖 HTML 结构；API 引擎未配置 Key；解析正则偏窄。
- 处理：默认 Bing 优先，DuckDuckGo 兜底；API 引擎默认关闭；增强 HTML 解析和宽松兜底。
- 状态：已优化，仍需设备端继续测试。

### 5. 远程规则不是启动自动加载

- 现象：新安装后主要依赖内置规则；远程规则需要手动更新。
- 原因：远程规则更新入口在设置页。
- 建议：后续改为启动后静默加载远程规则，失败时继续使用内置规则。
- 状态：待实现。

### 6. `Index.ets` 单文件过大

- 现象：页面、状态、搜索、阅读、设置全部集中在一个文件，在线局部修改困难。
- 原因：早期快速迭代未拆分页面和组件。
- 处理：后续按页面和组件逐步拆分。
- 状态：待重构。

### 7. GitHub 文件接口不适合修改大文件

- 现象：在线工具只能整文件替换，修改 `Index.ets` 风险高。
- 处理：固定使用一个 workflow + `scripts/polish_ui.py` 进行确定性文本修改，再固化到 `develop`。
- 状态：已建立临时流程规范，后续需保持脚本幂等。

### 8. Workflow 不能直接针对 `main`

- 现象：基础测试 checkout `main`，与开发流程冲突。
- 需求：所有自动流程不能针对主分支运行。
- 处理：基础测试、合规检查改为 develop；固定 UI workflow 改为手动选择目标分支并禁止 main。
- 状态：已修正。

### 9. 基础测试 workflow 结构检查失败

- 现象：`检查 workflow 关键结构` 步骤失败。
- 原因：检查 `.github/workflows/manual-build-entry.yml`，但实际文件为 `.github/workflows/manual-build-entry-advanced.yml`。
- 处理：更新检查文件名和流程名称。
- 状态：已修正。

### 10. Workflow YAML 中嵌入大段 Python 导致识别异常

- 现象：GitHub Actions 页面显示/识别不正常，疑似缩进问题。
- 原因：workflow 中直接嵌入三引号 Python 和大段 ArkTS 文本。
- 处理：把复杂逻辑移动到 `scripts/polish_ui.py`，workflow 只调用脚本。
- 状态：已修正。

### 11. 清理流程默认行为需要更安全

- 现象：清理流程需要支持预览和清空 workflow run 记录。
- 需求：默认预览；可选清理所有 workflow run 运行记录，默认保持清空状态不执行删除。
- 处理：新增 `clear_all_workflow_runs`，默认 `false`；`dry_run` 默认 `true`。
- 状态：已修正。

### 12. 文档散落缺少统一规范

- 现象：需求、问题、规范、构建说明、流程说明散落在 README、对话和 workflow 中。
- 处理：新增 `docs/` 文档中心和开发规范文档。
- 状态：进行中。

### 13. UI 自动脚本留下旧 SearchHome Builder 残留

- 现象：`Index.ets` 中出现 `LegacySearchHomeRemoved`、`SearchHomeOldUnusedAnchor` 等旧 Builder 残留，代码结构不干净，可能导致后续构建或维护问题。
- 原因：早期脚本通过锚点替换大段 `SearchHome()`，未完整删除旧块尾部。
- 处理：固定 workflow 改为运行目标分支自己的 `scripts/polish_ui.py`；脚本改为保守幂等清理，检测到旧残留则直接失败或清理。
- 状态：脚本已更新，待运行固定 workflow 固化到 `Index.ets`。

### 14. 书架页仍显示收藏和历史

- 现象：需求要求书架不放收藏内容，但代码中的 `ShelfPage()` 仍显示收藏网格、阅读历史、清历史、清书架。
- 原因：早期书架页承担收藏和历史入口，后续 Tab 信息架构调整后未完全迁移。
- 处理：固定 UI 脚本改为把清历史、清收藏、收藏打开、收藏移除等现有功能迁移到历史页；书架页改为热门题材推荐入口。
- 状态：脚本已更新，待运行固定 workflow 固化到 `Index.ets`。

### 15. 书架定位调整为热门题材推荐

- 现象：原书架页定位仍偏收藏管理，不符合后续产品方向。
- 原因：产品信息架构调整为“搜索 / 书架 / 历史 / 设置”后，书架 Tab 需要承载发现入口。
- 处理：书架页改为热门题材推荐，后续从规则仓库或目录索引自动更新；当前先提供内置题材卡片并点击搜索。
- 状态：脚本和文档已更新，待运行固定 workflow 固化到 `Index.ets`。

### 16. APP 主图标需要使用 PNG

- 现象：APP 桌面显示图标和搜索首页图形需要使用 PNG，不能继续依赖同名 SVG。
- 原因：HarmonyOS / OpenHarmony 应用图标资源使用 PNG 更稳；用户要求直接使用沟通预览图中的图形扣图。
- 处理：新增 `scripts/generate_app_icon_png.py`，从已确认预览图扣出的 PNG base64 生成 `entry/src/main/resources/base/media/icon.png`，并删除 `icon.svg` 避免 `$media:icon` 资源名冲突；固定 UI workflow 已增加 PNG 图标生成步骤。
- 状态：脚本和 workflow 已更新，待运行固定 workflow 固化到资源目录。

### 17. 搜索结果卡片和标题过长

- 现象：搜索结果列表/网格不够明确，列表高度偏大；部分标题重复显示导致卡片名称很长。
- 原因：早期结果卡片混入描述和按钮，网格/列表逻辑还保留两列切换；搜索源返回标题可能包含重复名称或重复分隔片段。
- 处理：固定 UI 脚本统一结果显示：列表为低高度左图右文横条；网格固定三列图片卡片；新增显示层标题清洗函数，去掉连续重复词、重复分隔片段和整段重复。
- 状态：脚本和文档已更新，待运行固定 workflow 固化到 `Index.ets`。
