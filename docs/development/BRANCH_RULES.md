# 分支规则

## 分支定位

### `main`

- 稳定分支。
- 只接收已经在 `develop` 测试确认的内容。
- 禁止作为自动测试、自动 UI 应用、自动清理等流程的直接目标。
- 不直接开发。

### `develop`

- 主要开发分支。
- UI、搜索、规则、设置、阅读器等改动优先进入该分支。
- 用户构建安装测试通过后，再合并到 `main`。

### `feature/*`

- 新功能分支。
- 适合拆分页面、下载功能、规则自动更新、搜索链路增强等较大改动。

### `fix/*`

- 修复分支。
- 适合构建失败、运行异常、UI 错位、搜索无结果等问题。

## 合并规则

- `feature/*` / `fix/*` → `develop`
- `develop` → `main`
- 不允许 `main` → 直接开发提交作为正常开发流程。

## Workflow 分支规则

- 自动触发的 `push` / `pull_request` 只允许针对 `develop` 或开发分支。
- 固定 UI 自动应用 workflow 必须拒绝 `main` 作为目标分支。
- 清理 workflow 只允许手动触发，默认预览。

## 测试规则

- 修改 workflow 后必须确保基础测试能识别正确 workflow 文件。
- 修改 UI 后必须至少构建安装一次。
- 修改搜索链路后必须测试中文、英文关键词。
