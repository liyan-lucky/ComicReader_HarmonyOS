# 开发需求记录

本文档记录已经确认的产品和开发要求。后续开发必须以本文档为准；如果需求变化，先更新本文档，再修改代码。

## 一、开发原则

- 当前唯一长期主分支是 `main`。
- 小范围文档、配置和低风险修复可直接进入 `main`。
- 风险较大的功能、结构调整或批量重构，应先从 `main` 新建 `feature/*` 或 `fix/*` 分支。
- 所有 GitHub Actions 自动流程不得无提示大规模修改源码。
- 禁止重新引入自动 UI 注入脚本、自动 patch 脚本或临时 UI 自动应用 workflow。
- 如果必须新增 workflow，先说明用途，并优先删除或替换旧临时 workflow。
- 修改必须尽量固化到实际源码文件，不长期依赖补丁说明。
- 分支、备份、构建、发布、能力边界变化时，必须同步更新 `docs/CURRENT_STATUS.md`、`README.md` 和相关专项文档。

## 二、APP 定位

- 项目名称：漫画浏览器。
- 重点能力：搜索、浏览、阅读。
- 不定位为单纯收藏工具。
- 不托管漫画图片、章节正文、付费内容、账号数据、站点 Logo 或字体。
- 只处理公开可访问资源；不实现登录绕过、付费绕过、验证码绕过、DRM 绕过或反爬规避。

## 三、底部 Tab

最终结构：

1. 搜索
2. 书架
3. 历史
4. 设置

要求：

- `搜索` 为默认 Tab。
- `关于` 不再作为独立 Tab。
- `关于` 信息整合到 `设置` 页面。
- `书架` 不再展示收藏列表，转为热门题材推荐入口。

## 四、搜索页

搜索首页要求：

- 只保留搜索功能。
- 页面构图按照预览图方向：上方/中部 APP 图标，中间一个搜索框。
- 搜索框上方图形直接使用沟通中预览图扣出的 PNG 图标。
- 不显示任何设置入口。
- 不显示搜索模式、来源过滤、主题、语言、演示按钮等设置类内容。
- 不显示复杂流程说明卡片。

搜索结果要求：

- 搜索后显示结果。
- 搜索结果只分为“列表”和“网格”两种显示形式。
- 默认使用三列图片网格。
- 网格卡片固定三列，每个卡片包含图片和名称。
- 列表卡片使用低高度横条样式，小图片在前，名称在后。
- 列表名称最多显示两行，超出省略。
- 排列切换功能只在有搜索结果时显示。
- 搜索结果标题需要做显示层清洗，去除连续重复名称、重复分隔片段和整段重复，避免标题变得很长。
- 搜索结果按来源展示，对应多个资源。
- 搜索默认无需 API Key 可用。
- 默认优先 Bing、DuckDuckGo 等无 Key 搜索链路。
- API Key 类能力放入设置页高级区。

## 五、书架 / 热门题材

- 书架 Tab 保留，但页面定位改为“热门题材推荐”。
- 书架页不展示收藏内容、不展示阅读历史、不提供清历史/清收藏。
- 书架页后续从规则仓库或公开目录的热门题材索引自动更新。
- 当前先提供常用题材入口，例如玄幻、修真、恋爱、校园、宫斗、都市、热血、冒险、悬疑、科幻、搞笑、治愈。
- 点击题材后跳转搜索页并按该题材搜索。

## 六、历史页

历史页接收原书架页中的已有功能，包含：

- 观看历史 / 阅读历史。
- 收藏记录。
- 下载。
- 清空历史。
- 清空收藏。
- 收藏记录打开。
- 收藏记录移除。

下载功能暂时保留，后续再接入离线章节或任务管理。

## 七、设置页

设置页负责整合所有设置和关于信息。

建议分区：

- 基础设置：主题、语言、阅读悬浮控制。
- 搜索设置：搜索模式、搜索语言、搜索引擎、额外关键词、站点提示。
- 公开源开关：公开 API、HTML 规则源、搜索发现页等。
- 显示与阅读：结果列数、列表样式、阅读器背景、图片适配、页码、去重、历史保留。
- 搜索引擎清单：启用/停用、当前搜索引擎。
- 远程规则：GitHub Raw 规则地址与更新。
- 高级规则：自定义 JSON 规则。
- 关于：版本、构建信息、目标平台、构建时间。

## 八、UI 风格

- 全局强调色：`#34C759`。
- 所有选中态使用绿色风格。
- Tab 选中时图标和文字绿色，不使用大面积绿色填充。
- 关闭、停用、禁用状态使用灰色，不使用绿色。
- APP 桌面图标、启动图标、搜索首页上方图形统一使用 PNG。
- APP PNG 图标来源为沟通中预览图扣图，资源路径为 `entry/src/main/resources/base/media/icon.png`。
- 不再使用同名 `icon.svg`，避免 `$media:icon` 资源冲突。
- Tab、设置、历史、阅读控制等功能图标继续使用统一 SVG 线性风格。

## 九、规则系统

- APP 启动默认有内置规则。
- 远程规则来自 `ComicReader_Rules` 仓库生成的 `generated/index.json`。
- 远程规则失败时保留内置规则。
- 自定义规则属于高级功能。
- 远程规则和自定义规则必须校验 schema、URL、数量和字段长度。
- 热门题材推荐后续应由规则仓库或目录索引自动生成并更新。
