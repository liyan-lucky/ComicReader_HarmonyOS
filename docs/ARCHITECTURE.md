# 架构说明

## 项目定位

漫画浏览器是 HarmonyOS / OpenHarmony ArkTS Stage 应用，核心目标是搜索公开漫画资源、整理结果并进入卷轴阅读。

## 页面结构

目标 Tab：

- 搜索：默认入口，只负责搜索。
- 书架：热门题材推荐入口，后续自动更新推荐题材。
- 历史：阅读历史、收藏记录、下载，以及原书架页中的清理/打开/移除管理能力。
- 设置：所有设置和关于信息。

## 数据流

```text
用户输入关键词 / 点击热门题材
  ↓
搜索链路
  ↓
公开 API / 搜索引擎 / HTML 规则源
  ↓
结果去重、过滤、归类
  ↓
封面结果网格或列表
  ↓
章节页 / 阅读页
  ↓
历史记录 / 收藏记录
```

## 热门题材推荐流

```text
规则仓库热门题材索引 / 内置默认题材
  ↓
书架 Tab 展示题材卡片
  ↓
点击题材
  ↓
切换到搜索页并自动搜索
```

当前先使用内置默认题材：玄幻、修真、恋爱、校园、宫斗、都市、热血、冒险、悬疑、科幻、搞笑、治愈。

后续目标是从规则仓库生成的热门题材索引自动更新。

## 搜索链路

默认无需 API Key：

1. Bing 网页搜索。
2. DuckDuckGo HTML 搜索。
3. Google / Yandex 作为补充。
4. Brave / Google CSE 放入高级设置，需要 Key。

结果处理：

- 标题清洗。
- URL 去重。
- 登录/付费/商城/百科/视频/社区结果过滤。
- 按名称归类。
- 自动补封面。

## 规则系统

规则来源：

1. 内置 `GeneratedSourceRules.ets`。
2. 内置 `SourceRules.ets`。
3. GitHub 远程规则 `generated/index.json`。
4. 用户自定义 JSON 规则。
5. 后续热门题材推荐索引。

规则加载原则：

- APP 默认必须有内置规则可用。
- 远程规则更新失败不影响内置规则。
- 远程规则必须校验 schema、数量、URL 和字段长度。
- 热门题材索引不能包含危险链接或非公开内容。

## 阅读流程

```text
搜索结果
  ↓
打开详情页
  ↓
解析章节
  ↓
解析章节图片
  ↓
卷轴阅读
  ↓
记录历史 / 收藏
```

如果静态 HTML 拿不到图片，可使用渲染兜底，但不得绕过登录、付费、验证码或 DRM。

## 当前技术债

- `Index.ets` 仍然过大。
- 页面、搜索、阅读、设置逻辑应逐步拆分。
- 远程规则建议后续启动后静默同步。
- 下载功能暂时保留，尚未完整实现任务管理。
- 热门题材推荐目前为内置默认题材，后续需要接入规则仓库自动生成索引。
